# 摸金虾 系统架构

> v1.0.1 · 2026-06-03

---

## 一、三体联动流水线

```
09:00 盘前定调 → 写入 market_context（主线/板块/偏向）
14:20 引擎一   → 写入 candidate_pool.engine1（STAGE2/3）
14:25 引擎二   → engine2_ranker.py → 交叉排名 → candidate_pool.engine2.ranked
15:05 收盘复盘 → 读 ranked + market_context → 生成 scenarios
```

**排名权重**：

| 权重 | 条件 |
|:---:|------|
| +3 | 引擎一 STAGE3 满分 |
| +2 | 引擎一 STAGE3 或 引擎二 Top10%+ |
| +2 | 匹配主线（market_context.main_themes） |
| +1 | 引擎一 STAGE2 或 引擎二 Top15 |

P0（权重≥5）= 双引擎+主线命中，最高优先级。

## 二、CRON 分层

```
策略层：收盘复盘（T-1 创建蓝图）+ 盘前分析（T 日微调）
侦察层：午后检查（13:00 信息最丰富）
执行层：午盘检查 + 尾盘检查（只读侦察快照 → 硬约束 → 匹配执行）
```

### 交付格式

| 时间 | CRON | 格式 | 说明 |
|------|------|:--:|------|
| 09:00 | 盘前定调 | 📰报纸 | 开盘前全景锚点 |
| 10:00 | 午盘执行 | 📝≤5行 | 风险+仓位+操作摘要 |
| 13:16 | 午后侦察 | 📝短文本 | 市场快照+预警 |
| 14:25 | 引擎二排名 | 📰报纸 | 排名结果+交叉命中 |
| 14:35 | 尾盘执行 | 📝文本 | 操作+持仓+逻辑 |
| 15:05 | 收盘复盘 | 📰报纸 | 全天最高信息密度 |

## 三、架构决策记录（ADR）

### ADR-001: 策略文件作为唯一数据总线

**日期**: 2026-05-28 · **状态**: ✅ 已采纳

每日独立策略文件（`strategies/{date}.json`）作为所有 CRON 间的唯一数据交换媒介。`strategy_index.json` 作为索引指针。

> 否决替代方案：`context_from` 注入（旁路、非结构化、不可审计）、共享数据库（过度设计）。

### ADR-002: risk_state 确定性计算

**日期**: 2026-05-28 · **状态**: ✅ 已采纳

`calc_risk_state.py` 硬编码五级规则（normal→cautious→defensive→emergency），所有 CRON 必须调用此脚本，禁止 LLM 手动覆盖。

**教训**：2026-06-02 事故 — CRON prompt 缺少「禁止 LLM 覆盖」规则，导致 `consecutive_loss_days` 跨交易日不归零，熔断误持续 2 天。已加固。

### ADR-003: 侦察收敛到午后单节点

**日期**: 2026-06-01 · **状态**: ✅ 已采纳

午盘和尾盘独立侦察逻辑复制两份 → 午后 13:16 集中侦察写结构化 `market_snapshot`，执行层只读快照 + 一次 API 验证。侦察从 4 次/天降至 3 次。

### ADR-004: 引擎分离，策略文件连接

**日期**: 2026-06-02 · **状态**: ✅ 已采纳

引擎一 → 写入 `candidate_pool.engine1`。引擎二 → 读取 engine1 → 写入 `candidate_pool.engine2.ranked`。尾盘/收盘从策略文件读取候选池。`context_from` 全部清零。

### ADR-005: 报纸只用于锚点时刻

**日期**: 2026-06-02 · **状态**: ✅ 已采纳

报纸图片仅用于两个锚点：开盘前（盘前早参）和收盘后（收盘复盘）。盘中全部用精简文本。

### ADR-006: 设计文档与操作手册分离

**日期**: 2026-06-02 · **状态**: ✅ 已采纳

`moni-sim-trading` skill 中 `references/design-*.md` 为产品设计文档，不带前缀的为操作参考。SKILL.md 顶部明确两条阅读路径。
