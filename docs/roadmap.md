## 迭代机制

### 三层节奏

| 层级 | 频率 | 触发 | 产出 | 规则 |
|------|------|------|------|------|
| **盯盘** | 每日 15:05 后 | 收盘复盘结束 | issues 记录（不改代码） | 盘中 09:00-15:05 不修改任何 CRON |
| **审计+排期** | 每周六 | 定时或手动 | 计分卡 + Scenario 审计 + 排期修复 | 集中改，改完跑验证 |
| **大版本** | 每月末 | 月目标核算 | 月复盘 + roadmap 刷新 + 版本号 | 改 3+ CRON → 版本号 +0.1 |

### 变更安全机制

- 盘中不改（09:00-15:05）
- 改完必跑验证（`cronjob run` 或样图）
- 双写：改 CRON prompt 同时更新 moni-sim-trading skill
- 回滚标注：changelog 用 `[REVERTED]` 而非删除
