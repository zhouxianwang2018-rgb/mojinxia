# 引擎一动态候选池

> SRS-002 · 创建：2026-06-03 · 状态：设计完成，待排期

---

## 一、业务需求

### 问题

引擎一（早期突破三阶段扫描）使用硬编码固定观察池（`DEFAULT_WATCHLIST`，6板块~55只）：

| # | 问题 | 真实案例 |
|:--:|------|------|
| 1 | **冷门板块异动全漏** | 力量钻石（培育钻石）5/7异动+20%，不在池中，10日涨幅+66% |
| 2 | **手动维护不可持续** | 上次更新已过2周，CPO二波等新主线未覆盖 |
| 3 | **无淘汰机制** | 异动失效率高的票反复消耗扫描资源 |

### 目标

观察池从"人工定期更新"变为"全市场自动流入+自然淘汰"——引擎一覆盖范围不再受限于人工维护的列表。

### 成功指标

| # | 指标 | 当前 | 目标 | 衡量方式 |
|:--:|------|:--:|:--:|------|
| 1 | 日异动覆盖率 | 仅扫描6板块 | **全市场** | `daily_inflow.py` 发现数 vs 全市场实际+8%数 |
| 2 | 重大遗漏 | 力量钻石漏掉 | **0 遗漏** | 凡当日+8%且量>1.5x都在池中 |
| 3 | 池规模 | 固定55只 | **50~200只**动态 | `dynamic_pool.json` 实时规模 |
| 4 | 淘汰率 | 0（无淘汰） | **每周≥20%自然淘汰** | 20日无进展自动移除 |
| 5 | 维护成本 | 每2周手动 | **0** | 无人工干预 |

## 二、方案设计

### 2.1 数据流

```
15:10 收盘后 ──→ engine1_pool_manager.py ──→ engine1_pool.json
                  │                              │
                  │ mx选股器                      ├─ 新STAGE1入池
                  │ 全市场+8%+量>1.5x             ├─ 已有标的更新状态
                  │                              ├─ 淘汰标记
                  └──────────────────────────────┘
                                                        │
14:20 引擎一扫描 ←── early_breakout_scanner.py ←────────┘
                        读取 engine1_pool.json
                        跑三阶段检测
                        输出STAGE2/STAGE3候选
```

### 2.2 池生命周期

```
        每日收盘流入              跟踪中              自然淘汰
    ┌─────────────────┐    ┌──────────────┐    ┌──────────────┐
    │ 全市场+8%+量>1.5x │───→│ status=active │───→│ 移除条件满足  │
    │ 概念主线补充(周)   │    │ 每日更新      │    │ → status=grave│
    └─────────────────┘    │ stage跟踪     │    └──────────────┘
                           └──────────────┘
                                  │
                           STAGE3触发 → 输出候选供尾盘使用
```

### 2.3 淘汰规则

| 条件 | 动作 |
|------|:--:|
| 异动后累跌 > 异动日开盘价 5% | `status=dead`，从扫描中移除 |
| 入池后 20 个自然日无 STAGE2/STAGE3 | `status=stale`，不再扫描 |
| STAGE3 触发后 5 日未执行 | 保留在池但不输出（避免重复推荐） |
| 触发过 STAGE3 且已清仓 | `status=harvested`，保留历史记录 |

### 2.4 周度概念补充

每周六跑概念主线扫描：
- 取本周涨幅 TOP5 概念板块
- 各取成交量 TOP3 标的
- 不在池中的补充入池，标记 `source=concept_supplement`

### 2.5 文件布局

```
~/.hermes/cron/state/moni/
├── engine1_pool.json              ← 新增：引擎一动态观察池
└── strategies/ ...

~/.hermes/scripts/
├── early_breakout_scanner.py      ← 改造：读 engine1_pool.json，不做池管理
└── engine1_pool_manager.py        ← 新增：池管理（流入+淘汰+周度补充）
```

**职责边界**：

| 脚本 | 做什么 | 不做什么 |
|------|------|------|
| `engine1_pool_manager.py` | 每日流入、淘汰判定、周度概念补充、写 `engine1_pool.json` | 不跑三阶段检测 |
| `early_breakout_scanner.py` | 读 `engine1_pool.json`，跑三阶段检测，输出候选 | 不改池文件 |

### 2.6 engine1_pool.json 数据结构

```json
{
  "updated": "2026-06-03T15:10:00",
  "stocks": {
    "300975": {
      "name": "商络电子",
      "source": "daily_inflow",
      "first_seen": "2026-06-03",
      "status": "active",
      "anomalies": [
        {
          "date": "2026-06-03",
          "gain": 12.3,
          "vol_ratio": 2.1,
          "open_price": 28.50,
          "position_pct": 35.0
        }
      ],
      "last_scan": {
        "date": "2026-06-03",
        "stage": "STAGE1"
      }
    }
  }
}
```

**字段说明**：

| 字段 | 类型 | 说明 |
|------|:--:|------|
| `stocks.{code}` | object | key 即股票代码，对象内不重复存 code |
| `name` | string | 股票名称 |
| `source` | enum | `daily_inflow` / `concept_supplement` / `seed` |
| `first_seen` | date | 入池日期，`days_in_pool` 由此推算 |
| `status` | enum | `active` / `dead` / `stale` / `harvested` |
| `anomalies` | array | 异动记录列表，一只票可多次异动 |
| `last_scan` | object | 最近一次三阶段扫描结果（快照） |
| `last_scan.stage` | enum | `STAGE1` / `STAGE2` / `STAGE3` |

**设计原则**：
- 生命周期字段（`status` / `first_seen` / `source`）与扫描快照（`last_scan` / `anomalies`）分离
- 不存可推导字段（`total_active` 由计数得出，`days_in_pool` 由 `first_seen` 推算）
- 不做预留空壳（不需要的先不加，用到了再扩展）

### 2.7 关键决策

| 决策 | 选择 | 原因 |
|:-----|:-----|:-----|
| 流入数据源 | mx-stocks-screener | 自然语言一行搞定，不遍历5000只 |
| 池上限 | 200只硬顶 | 保证扫描耗时可控（~10秒） |
| 风险兜底 | mx挂了用上一日池 | 不发空池，避免引擎一哑火 |
| 冷启动 | 旧池作种子 + 近5日异动回填 | 不空池起步 |
| 池管理 | 单脚本 `engine1_pool_manager.py` | 流入/淘汰/周度补充归一口径，避免多脚本写同一文件 |

## 三、风险与缓解

| 风险 | 概率 | 缓解 |
|------|:--:|------|
| mx选股器返回超时/空 | 中 | 重试3次，仍失败则用上一日池继续扫描 |
| 流入过多（单日>50只异动） | 低 | 上限50只/日，按量比排序取前50 |
| 牛市池膨胀到500+ | 中 | 20日淘汰+跌幅淘汰自然收缩，硬上限200只 |
| 旧扫描器改崩 | 中 | 先做副本 `early_breakout_scanner_v2.py`，验证后替换 |
