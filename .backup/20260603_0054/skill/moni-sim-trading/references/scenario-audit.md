# Scenario 审计

## 工具

```bash
python3 ~/.hermes/scripts/scenario_audit.py              # 全量审计
python3 ~/.hermes/scripts/scenario_audit.py --latest 5   # 最近5天
python3 ~/.hermes/scripts/scenario_audit.py --json       # JSON输出
```

输出 markdown 报告：触达率、攻防比、死亡条件列表、重复检测、按类型触达率。

## 首次审计发现（2026-06-01，3个交易日）

### 核心数据

| 指标 | 值 |
|------|-----|
| 总 scenarios | 14 |
| 触达 | 3 (21%) |
| 进攻型触达 | 0/5 |
| 防御型触达 | 2/5 |
| 重复组 | 5组（5/28和5/29全量复制） |

### 三病灶

1. **「反弹至 X 价格」是死条件** — 5个 reduce 型全部基于绝对价格（如"中微回升至470+"），在下跌市中形同虚设
2. **5/28→5/29 全量复制** — 两组 scenario 完全相同，复盘继承时未根据市场变化调整条件
3. **进攻型 0 触达** — 不是因为市场没给机会（5/28 长电逆势+1.91%），而是缺少进攻型入口

### 触达率按类型

| 类型 | 总数 | 触达 | 触达率 |
|------|:----:|:----:|:------:|
| 🛑 熔断 | 3 | 2 | 67% |
| ⏸️ 持有 | 3 | 1 | 33% |
| 🛡️ 止损 | 2 | 0 | 0% |
| 📉 减仓 | 5 | 0 | 0% |

### 改进方向

- P0: 绝对价格触发 → 相对指标（"连续2日未创新低"/"站上5日线"）
- P1: 复盘继承 scenario 时必须调整条件，不允许原文复制
- P2: 增加进攻型 scenario，攻防比从 5:5 调到至少 3:5

## 与计分卡联动

- 进化维度的 Scenario质量 KPI 追踪触达率和重复率
- 触达率 <30% → 计分卡自动标记 `⚠️ Scenario触达率仅X%，大量死条件 → 需审计`
- 建议每周六跑一次 `scenario_audit.py`，嵌入车牌交叉分析 cron 之后
