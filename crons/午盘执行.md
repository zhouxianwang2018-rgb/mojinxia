## Step -1: 前置安全校验

```bash
TODAY=$(date +%Y-%m-%d)
STRATEGY_FILE=~/.hermes/cron/state/moni/strategies/$TODAY.json

if [ ! -f "$STRATEGY_FILE" ]; then
  echo "🔴 终止：$TODAY 不是交易日。"
  exit 0
fi

STATUS=$(python3 -c "import json; print(json.load(open('$STRATEGY_FILE'))['status'])")
if [ "$STATUS" = "closed" ]; then
  echo "🔴 终止：策略已关闭。"
  exit 0
fi

HAS_ALREADY=$(python3 -c "
import json
with open('$STRATEGY_FILE') as f:
    d = json.load(f)
logs = [e for e in d.get('execution_log', []) if e.get('cron') == '午盘检查']
print(len(logs))
")
if [ "$HAS_ALREADY" -gt 0 ]; then
  echo "🔴 终止：已有 $HAS_ALREADY 条午盘记录。"
  exit 0
fi

echo "✅ 安全检查通过"
```

## Step 0a: 硬约束检查（按顺序，不可绕过）

```
LEVEL = risk_state.level
HARD_CAP = risk_state.hard_position_cap
BAN = risk_state.new_position_ban
LOSS_DAYS = risk_state.consecutive_loss_days
```

规则1: LEVEL=="emergency" → 🔴 全面清仓，终止
规则2: 仓位 > HARD_CAP → 🔴 强制降仓到上限以下（优先砍浮亏最大的）
规则3: BAN==True → 🔴 禁止任何买入（即使盘前 override 也不行）
规则4: LOSS_DAYS>=2 → 🔴 单票上限减半，禁止对浮亏加仓


## 错误处理规则（全局）
- 任何 API 调用失败 → 重试一次，仍失败 → 跳过该步骤，在 execution_log 中记录 "api_error: <步骤名>"
- moni_engine.py 不可用 → 尝试直接 httpx/curl 调用妙想 API
- 一个环节失败不终止整个 CRON，尽最大努力完成其他步骤

⚠️ 硬约束(0a)优先级最高，不可绕过。
